4.2 Optional类

Optional类提供了对值的封装，并且该值可以为null。并且提供了根据值是否是null来实现不同业务逻辑的方法。

4.2.1 创建Optional实例

可以调用Optional的of系列方法来创建Optional实例，这些方法包括：

  • of(value):创建封装value的Optional实例。value不能为null，否则抛出NullPointerException异常。
  • ofNullable(value):创建封装value的Optional实例。value可以为null。

4.2.2 判断封装的值是否是null

判断封装的值是否是null的方式十分简单，调用Optional实例的isPresent()方法就可以了。

4.2.3 获取被封装的值

调用Optional实例的get()方法。如果有值，则返回该值。如果被封装的是null，则抛出NoSuchElementException。

4.2.4 根据被封装的是否为null来进行处理

Optional中提供了一系列的方法来根据被封装的值是否为null来进行逻辑处理，包括：

  • ifPresent(consumer): 该方法的参数是一个Consumer接口的实例（我们可以使用Lambda来实现）。如果被封装的值不为null，则使用consumer来处理该值;如果被封装的值为null，则什么也不做。

  • orElse(defValue):如果Optional中封装的值不是null，则返回被封装的值；否则返回传入的defValue。

  • orElseGet(supplier):如果Optional中封装的值不是null，则返回被封装的值；否则，返回调用supplier得到的值。

  • orElseThrow(supplier):如果Optional中封装的值不是null，则返回该值；否则抛出supplier创建的异常。

4.2.5 对Optional的其他操作

  • map(function):该方法的逻辑如下：

    • 如果Optional中的值为null，则返回一个空的Optional的。
    • 如果Optional中的值不是null，则使用function处理该值。如果得到的值为null，则返回一个空的Optional。如果得到的值不是null，则返回一个封装了结果的Optional。

  • flatMap(function):该方法和map方法的逻辑一样，不同的是map中的function可以返回任何类型的值，map方法会将该值封装为Optional；但是在flatMap中，我们的function中需要自行将值封装为Optional。

4.3.1 SecureRandom

SecureRandom不能指定种子，而是使用random number generator来生成。JDK中的SecureRandom有多种不同强度的实现，在实际使用的时候，可以优先获取高强度的安全随机数生成器，如果没有，再考虑使用普通等级的。
